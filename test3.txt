Skipped
A user has left the analytics team and should no longer be able to query the table prod.finance.invoices, on which the user was previously granted SELECT directly.

Which SQL command correctly removes that user's read access?

ALTER TABLE prod.finance.invoices REMOVE READER priya@example.com;

DENY SELECT ON TABLE prod.finance.invoices FROM priya@example.com;

DROP USER priya@example.com FROM TABLE prod.finance.invoices;

Correct answer
REVOKE SELECT ON TABLE prod.finance.invoices FROM priya@example.com;

Overall explanation
REVOKE is the standard Unity Catalog command to remove a previously granted privilege. The pattern is REVOKE privilege ON object FROM principal. DROP USER and ALTER TABLE REMOVE READER are not Unity Catalog grant commands. DENY is not part of Unity Catalog's privilege model; the practical equivalent is to revoke and not regrant. As a side note, the broader best practice is to manage access via groups so removal happens via group membership.
